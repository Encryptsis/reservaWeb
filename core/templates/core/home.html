Backend listo para crecer con tu operación

Ya puedes registrar reservas, revisarlas en el administrador de Django y usar esta base para añadir confirmaciones, reglas de ocupación, mesas y recordatorios.

Persistencia Las solicitudes se guardan en SQLite mediante un modelo `Reservation`.
Validación El formulario usa `ModelForm` para validar correo, fecha, hora y tamaño del grupo.
Panel admin El equipo puede revisar y filtrar reservas desde `/admin/`.
Siguiente paso Añadir disponibilidad real, confirmación por email y gestión de mesas.

Lo que ya resuelve esta versión

El flujo ya no es solo visual: el backend captura datos útiles de contacto y de la reserva, dejando lista una base sólida para la lógica operativa del restaurante.

Datos clave Nombre, email, teléfono, fecha, hora, comensales y peticiones especiales.
Estado inicial Cada solicitud se crea como pendiente para facilitar la confirmación posterior.
